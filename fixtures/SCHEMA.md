# Golden fixture & case schema

Two file kinds, both `<plugin>.json` keyed by plugin slug (`custom-json`,
`custom-form`, `custom-xml`, `apprise-api`):

- **`cases/<plugin>.json`** — hand-authored input. Drives the capture harness.
- **`fixtures/<plugin>.json`** — generated by `scripts/capture_fixtures.py`
  from the matching case file. The golden artifact the TS diff test asserts
  against. Do not hand-edit; regenerate.

Both are produced/consumed by `scripts/capture_fixtures.py` (see its header
for the adapter-layer capture rationale). Fixtures are captured faithfully:
headers/body are byte-for-byte what upstream apprise **v1.12.0** emits, with
no normalisation (normalisation is the TS diff side's job, task 5.3).

## `cases/<plugin>.json`

```jsonc
{
  "plugin": "custom-json",          // optional; defaults to filename stem
  "cases": [
    {
      "name": "default",            // unique within the file
      "url": "json://user:pass@localhost/path",
      "title": "hi",                // optional, default ""
      "body": "hello body",         // optional, default ""
      "type": "info",               // info|success|warning|failure, default info
      "attachments": [ /* descriptors, optional */ ],
      "seeds": {                    // optional; defaults applied per field
        "uid": "itest-uid-0",       // -> asset._uid  (apprise-api X-Apprise-ID)
        "recursion": 0,             // -> asset._recursion (X-Apprise-Recursion-Count = +1)
        "boundary": null            // multipart boundary pin (null = none this batch)
      }
    }
  ]
}
```

Defaults when a case omits `seeds` (or a field): `uid="itest-uid-0"`,
`recursion=0`, `boundary=null`.

### Attachment descriptors (deterministic only)

```jsonc
// file: fixed-content path relative to the subproject root
{ "file": "cases/assets/hello.txt" }

// memory: inline bytes. `name` is REQUIRED (else upstream auto-names with
// uuid4()+.txt/.dat which would leak a non-deterministic filename).
{ "memory": { "text": "the content" }, "mimetype": "text/plain",  "name": "note.txt" }
{ "memory": { "base64": "AAEC" },      "mimetype": "application/octet-stream", "name": "blob.dat" }
```

## `fixtures/<plugin>.json`

```jsonc
{
  "plugin": "custom-json",
  "generated_by": "scripts/capture_fixtures.py",
  "apprise_version": "1.12.0",
  "cases": [
    {
      "name": "default",
      "input": {                    // echoes the driving case
        "url": "json://user:pass@localhost/path",
        "title": "hi", "body": "hello body", "type": "info",
        "attachments": [ /* echoed if present */ ]
      },
      "seeds": { "uid": "itest-uid-0", "recursion": 0, "boundary": null },
      "expected": { /* exactly one of: */
        "request": {                // normal delivery
          "method": "POST",
          "url": "http://localhost/path",   // final, query already joined
          "headers": { "User-Agent": "Apprise", "Content-Type": "application/json",
                       "Authorization": "Basic ...", "...": "..." },
          "body": { "text": "..." } // see body encoding below
        }
      }
    },
    {
      "name": "invalid-method",
      "input": { "url": "json://localhost/path?method=BOGUS", "...": "..." },
      "seeds": { "...": "..." },
      "expected": {                 // construction rejected / no request emitted
        "noRequest": { "reason": "instantiation-failed" }  // or "no-request"
      }
    }
  ]
}
```

### `expected` — two mutually exclusive shapes

- **`request`** `{ method, url, headers, body }` — a normal delivery. `headers`
  is the full set upstream sent (transport defaults included); the TS diff test
  compares only its per-plugin key set and ignores transport-default keys.
- **`noRequest`** `{ reason }` — no wire request. `reason`:
  - `"instantiation-failed"` — upstream refused to construct the plugin
    (invalid `?method=`, bad token, ...). `.add()` returned False.
  - `"no-request"` — constructed fine but emitted nothing (e.g. empty content).
  The TS side asserts it likewise constructs-and-refuses / sends nothing.

### `body` encoding (self-describing)

- `null` — no body.
- `{ "text": "<str>" }` — str payload (JSON, url-encoded form).
- `{ "base64": "<b64>" }` — binary payload (multipart). Round-trips losslessly.
